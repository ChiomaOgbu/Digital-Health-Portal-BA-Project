## Project Summary

This project presents a user-centred digital health portal concept designed to improve healthcare access for remote and rural communities.

The solution integrates telehealth services, AI-assisted support, offline functionality, and streamlined consultation workflows to improve accessibility, reduce service gaps, and enable remote decision-making.



## Business Problem

Remote and rural communities experience:

- Limited healthcare access
- Delayed consultations
- Infrastructure and connectivity challenges
- Poor continuity of care

A scalable digital solution was required to strengthen equitable healthcare delivery.



## Project Objectives

- Improve access to healthcare services
- Reduce service gaps in underserved regions
- Enable remote clinical decision-making
- Design a scalable and technology-driven solution



## Proposed Solution

The digital health portal includes:

- Telehealth appointment scheduling
- Virtual consultations
- AI-assisted symptom triage
- Offline access for low-bandwidth regions
- Secure patient data management
- Quick consultation and referral workflows



## Stakeholder Analysis

Primary stakeholders include:

- Rural Patients
- Healthcare Providers
- Health Authorities
- IT & Support Teams

Each stakeholder group was analysed based on impact, influence, and system usage.



## Functional Requirements

- User registration and secure authentication
- Telehealth booking system
- AI symptom assessment tool
- Access to patient records
- Appointment notifications
- Offline data storage with sync capability

---

## Non-Functional Requirements

- High data security and privacy compliance
- 99% system availability target
- Low-bandwidth optimisation
- Scalable architecture
- Response time under 3 seconds (where possible)



## Sample User Stories

**As a rural patient**, I want to book a telehealth consultation so that I can access medical care without travelling long distances.

**As a healthcare provider**, I want access to patient history before consultation so that I can make informed decisions.

**As a user in a low-connectivity area**, I want offline functionality so that essential features remain accessible.
