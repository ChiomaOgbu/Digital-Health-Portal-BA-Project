# User Stories

## Patient Stories

**User Story 1**
As a rural patient, I want to book a telehealth appointment so that I can consult a doctor without travelling long distances.

Acceptance Criteria:
- User can select date and time.
- User receives booking confirmation.
- Appointment reminder is sent automatically.

---

**User Story 2**
As a patient, I want AI symptom guidance so that I can understand possible next steps before consultation.

Acceptance Criteria:
- User completes questionnaire.
- System generates symptom summary.
- Results are stored in patient profile.

---

## Healthcare Provider Stories

**User Story 3**
As a healthcare provider, I want access to patient history before consultation so that I can make informed decisions.

Acceptance Criteria:
- Doctor can view medical history.
- Uploaded documents are accessible.
- AI triage summary is visible.

---

## Offline User Story

**User Story 4**
As a user in a low-connectivity area, I want offline access so that essential features remain usable.

Acceptance Criteria:
- Core information is accessible offline.
- Data syncs automatically when connection is restored.
